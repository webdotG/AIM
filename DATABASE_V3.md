# Database Architecture Version 3

> Version 3 не заменяет Version 2.
>
> Она расширяет существующую архитектуру.
>
> Ни одна существующая сущность не удаляется.
>
> Добавляются только новые абстракции, необходимые для моделирования развития человека.

---

# Главная идея

База данных проектируется не как база дневника.

Не как база заметок.

Не как база привычек.

И даже не как база психологического анализа.

Version 3 строится как фундамент цифровой модели человека.

Мы не пытаемся воспроизвести работу мозга.

Мы лишь сохраняем объективные факты жизни человека и связи между ними.

Именно эти факты позволяют анализировать развитие личности сегодня и совершенствовать алгоритмы анализа завтра.

---

# Философия

Человек — это постоянно изменяющаяся система.

Мысли влияют на действия.

Действия влияют на мысли.

Сны влияют на планы.

Планы приводят к действиям.

Действия становятся воспоминаниями.

Воспоминания рождают новые мысли.

Любая сущность может стать причиной любой другой.

Человеческая жизнь не является линейной.

Она представляет собой постоянно растущий граф взаимосвязанных событий и объектов.

Version 3 предназначена именно для хранения этого графа.

---

# Архитектурные принципы

---

## Принцип №1. Факты неизменны

База данных хранит только объективные факты.

Например

✓ Пользователь пробежал 8.4 км.

✓ Потратил 46 минут.

✓ После пробежки записал мысль.

✓ Через несколько часов нарисовал рисунок.

✓ Ночью увидел сон.

Факты никогда не изменяются.

---

## Принцип №2. Интерпретации изменяемы

Любые выводы являются результатом анализа фактов.

Например

"Пользователь стал дисциплинированнее."

"Воображение значительно выросло."

"Регулярный бег уменьшает тревожность."

"Изучение английского развивает память."

Это не данные.

Это интерпретации.

По мере развития алгоритмов анализа они могут меняться.

---

## Принцип №3. Никогда не хранить вычисленные выводы

Version 3 намеренно НЕ хранит

+3 дисциплины

+5 памяти

+2 логики

+4 силы воли

База хранит только реальные действия человека.

Любые характеристики вычисляются на основании накопленной истории.

---

## Принцип №4. История важнее оценки

Сегодня алгоритм может считать дисциплину одним способом.

Через год появится более совершенный алгоритм.

Через пять лет анализ станет ещё точнее.

При этом вся история пользователя останется неизменной.

Изменится только способ её анализа.

---

## Принцип №5. Память отделена от интеллекта

Version 3 разделяет два уровня системы.

Memory Layer

• факты

• записи

• связи

• измерения

AI Layer

• анализ

• поиск закономерностей

• вычисление характеристик

• рекомендации

• прогнозы

База данных ничего не знает о дисциплине, таланте или интеллекте.

Она знает только факты.

Все выводы делает отдельный уровень анализа.

# Основные домены

Version 3 строится вокруг пяти независимых доменов.

---

# 1. Entity

Entity — всё, что существует.

Например

Dream

Thought

Memory

Plan

Action

Person

Book

Project

Place

Goal

Habit

...

Каждая сущность является полностью самостоятельной.

Мысль может существовать без причины.

План может никогда не реализоваться.

Сон может остаться ни с чем не связанным.

Действие может быть совершено спонтанно.

Version 3 сознательно НЕ использует единую таблицу Event.

Все сущности имеют собственную предметную область и собственную таблицу.

Связи между ними строятся отдельно.

---

# 2. Relation

Relation описывает смысловые связи между сущностями.

Например

Thought

→ inspired →

Action

Action

→ caused →

Thought

Dream

→ reminded →

Memory

Book

→ changed →

Thought

Plan

→ resulted_in →

Action

Любая сущность может быть связана с любой другой.

Связь является самостоятельной сущностью.

Она содержит

• тип связи

• силу связи

• степень уверенности

• источник создания (пользователь / AI / система)

В будущем Relation может быть расширен до гиперграфа, позволяющего связывать несколько сущностей одной логической связью.

---

# 3. Activity

Activity описывает виды деятельности.

Например

Бег

Английский

Чтение

Рисование

Музыка

Программирование

Медитация

Фотография

Activity не хранит историю пользователя.

Activity — это справочник.

Action — конкретный экземпляр Activity.

Например

Activity

↓

Бег

↓

Action

↓

Сегодня пробежал 6.8 км за 38 минут.

---

# 4. Measurement

Measurement описывает всё, что можно измерить.

Например

Расстояние

Время

Вес

Повторения

Подходы

Пульс

Количество слов

Страницы

Уровень понимания

Яркость сна

Осознанность сна

Важность мысли

Интенсивность эмоции

Version 3 не ограничивает список измерений.

Новые измерения могут появляться без изменения структуры базы данных.

Измерения являются универсальным механизмом хранения объективной информации.

---

Characteristic описывает качества личности.

Это НЕ действия.

Это НЕ привычки.

Это НЕ Activity.

Это свойства человека, которые постепенно изменяются под воздействием жизни.

Например

Физика

• ответственность

Творчество

• воображение

Characteristic представляет собой отдельное дерево.

Хранится только история действий.

Характеристики вычисляются алгоритмом анализа.

---

# Почему Action выделен отдельно

Action — единственная сущность, описывающая реальное взаимодействие человека с окружающим миром.

Именно действия соединяют внутреннюю психическую жизнь человека с внешней.

Мысль может привести к действию.

Но действие может породить новую мысль.

Сон может изменить план.

План может привести к действию.

Действие может полностью изменить мировоззрение.

Action не является центральной сущностью графа.

Он лишь представляет физическое взаимодействие человека с миром.

---

# Как развивается человек

Version 3 рассматривает развитие личности как накопление опыта.

Не существует одного действия, которое сразу увеличивает дисциплину или память.

Любое изменение личности является следствием множества факторов.

Например

Мысль

↓

План

↓

Регулярные пробежки

↓

Повышение уверенности

↓

Новая мысль

↓

Изучение английского

↓

Новые знакомства

↓

Изменение мировоззрения

↓

Рост личности

Таким образом база данных хранит не оценки человека, а историю его развития.

---

# Масштабируемость

Version 3 проектируется таким образом, чтобы новые сущности могли появляться без изменения существующей архитектуры.

Можно добавить

• новую сущность;

• новый вид Activity;

• новую характеристику;

• новое измерение;

• новый тип связи;

без изменения фундаментальной структуры базы данных.

Это позволяет системе развиваться годами без постоянных миграций и переписывания архитектуры.

---

# Главная цель Version 3

Создать фундамент не для дневника.

Не для трекера привычек.

Не для RPG.

Модели, которая позволяет хранить

• внутренний мир;

• внешний опыт;

• причинно-следственные связи;

• объективные измерения;

• историю развития личности.

Version 3 хранит жизнь человека.

Интеллект системы лишь помогает её понять.
