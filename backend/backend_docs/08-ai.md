# Module 8: AI

## Overview

The AI module provides interfaces to an external AI service for node content analysis and image generation. Requests are sent via `fetch()` to the external service (configured via `AI_SERVICE_URL`), and results are persisted in the database as immutable generation facts.

**Location:** `src/modules/ai/`

---

## Architecture

```
Request → AIController → AIAnalysisService → AIRepository (inline) → PostgreSQL
                                   ↓
                            global.fetch() → External AI Service
                                   ↓
                            NodesRepository (ownership check)
                            getNodeContent() → dreams/thoughts/memories/plans/actions/projects tables
```

### Components

| Component | File | Purpose |
|-----------|------|---------|
| `AIController` | `controllers/AIController.ts` | HTTP handlers — analysis and image endpoints |
| `AIAnalysisService` | `services/AIAnalysisService.ts` | Orchestration: content extraction, external API calls, persistence |
| `AIRepository` | `services/AIAnalysisService.ts` | Inline class — DB queries for ai_analysis and ai_images |

**Note:** `AIRepository` is defined inline within `AIAnalysisService.ts` alongside the service class, rather than in a separate file.

---

## Database Tables

### `ai_analysis` — Persisted analysis results

| Column | Type | Constraints |
|--------|------|-------------|
| `id` | INT | PK, SERIAL |
| `node_id` | UUID | FK → `nodes.id` ON DELETE RESTRICT |
| `analysis_type` | VARCHAR | Type label (e.g., `dream_interpretation`) |
| `ai_model` | VARCHAR | Model identifier, nullable |
| `prompt` | TEXT | Prompt sent to AI, nullable |
| `result` | TEXT | AI response text |
| `metadata` | JSONB | Additional metadata |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() |

### `ai_images` — Persisted generated images

| Column | Type | Constraints |
|--------|------|-------------|
| `id` | INT | PK, SERIAL |
| `node_id` | UUID | FK → `nodes.id` ON DELETE RESTRICT |
| `image_url` | VARCHAR | URL to generated image |
| `prompt` | TEXT | Prompt used, nullable |
| `metadata` | JSONB | Additional metadata |
| `ai_model` | VARCHAR | Model identifier, nullable |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() |

### ON DELETE RESTRICT
Both tables use `ON DELETE RESTRICT` on `node_id`. These records survive node soft-delete — they are **facts of generation**, not cache. The AI service was called, resources were consumed, and the result is a historical fact preserved independently of the node's lifecycle.

## External AI Service Protocol

### Analysis endpoint
**URL:** `${AI_SERVICE_URL}/analyze`
**Method:** POST
**Body:**
```json
{
  "node_id": "<uuid>",
  "node_type": "<type_code>",
  "content": "<extracted content>",
  "analysis_type": "<type>"
}
```
**Response:**
```json
{
  "model": "<string|null>",
  "prompt": "<string|null>",
  "result": "<string>",
  "metadata": "<any>"
}
```

### Image generation endpoint
**URL:** `${AI_SERVICE_URL}/generate-image`
**Method:** POST
**Body:**
```json
{
  "prompt": "<string>"
}
```
**Response:**
```json
{
  "model": "<string|null>",
  "prompt": "<string|null>",
  "result": "<string>",
  "metadata": "<any>",
  "image_url": "<string>"
}
```

AI service URL defaults to `http://localhost:8000` if `AI_SERVICE_URL` env var is not set.

---

## Content Extraction (`getNodeContent`)

The `getNodeContent(nodeId)` private method scans multiple tables to find content for a node:

1. **Content tables** (column `content`): `dreams`, `thoughts`, `memories`
2. **Description tables** (column `description`): `plans`, `actions`, `projects`

Uses a single `Pool` connection, iterates through tables in order, returns first non-null match. If no content found, returns empty string. Errors on individual table queries are silently caught (the node may not have a row in that table).

## Endpoints

All routes require authentication via `authenticate` middleware.

### Analysis

| Method | Path | Controller | Service | Description |
|--------|------|-----------|---------|-------------|
| POST | `/analysis/:nodeId` | `requestAnalysis` | `requestAnalysis()` | Request AI analysis |
| GET | `/analysis/:nodeId` | `getAnalysis` | `getAnalysis()` | Get existing analyses |

**POST workflow:**
1. Validate node ownership
2. Fetch node type from `NodesRepository`
3. Extract content via `getNodeContent()`
4. Call external AI service via `fetch()`
5. On success, persist result in `ai_analysis` table via `AIRepository`
6. Return persisted row

**GET:** Returns all analyses for a node, ordered by `created_at DESC`.

### Image Generation

| Method | Path | Controller | Service | Description |
|--------|------|-----------|---------|-------------|
| POST | `/image/:nodeId` | `requestImage` | `requestImageGeneration()` | Generate an image |
| GET | `/image/:nodeId` | `getImages` | `getImages()` | Get existing images |

**POST workflow:**
1. Validate node ownership
2. Determine prompt: `req.body.prompt` if provided, otherwise `getNodeContent(nodeId)`
3. Reject if content is empty (throw `ValidationError`)
4. Call external AI service via `fetch()`
5. Validate `image_url` is present in response
6. Persist in `ai_images` table via `AIRepository`
7. Return persisted row

**GET:** Returns all images for a node, ordered by `created_at DESC`.

## Error Handling

| Scenario | Response |
|----------|----------|
| Node not found / not owned by user | 404 `NotFoundError` |
| AI service returns non-OK | 400 `ValidationError` ("AI service error" / "AI image generation error") |
| No `image_url` in response | 400 `ValidationError` ("No image_url returned") |
| Empty content for image generation | 400 `ValidationError` ("Content required for image generation") |

## Design Notes

1. **Inline repository pattern:** `AIRepository` is defined in the same file as `AIAnalysisService` because it serves only this service and has no shared utility value.

2. **No caching layer:** Results are persisted on every request. The AI service is called each time POST is invoked — there is no deduplication or cache check.

3. **Immutable facts:** Once a result is stored, it cannot be modified or deleted through the API. This aligns with the architecture principle that "facts are immutable."

4. **Flexible analysis types:** The `analysis_type` field is a free-form string, allowing future analysis types without schema changes.

5. **Metadata storage:** Both tables use JSONB `metadata` columns for extensible additional data from the AI service.
