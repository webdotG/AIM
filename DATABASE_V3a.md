# ARCHITECTURE_V3.md

# Version 3 — Life Graph Architecture

> "моделируем историю жизни / личность человека."

---

# Введение

Version 3 является развитием Version 2.

Основная идея новой архитектуры заключается в том, что приложение перестает быть исключительно психологическим дневником.

Теперь оно представляет собой универсальную систему хранения и анализа человеческой жизни.

Version 2 хорошо описывала внутренний мир человека:

- сны;
- мысли;
- воспоминания;
- планы;
- эмоции;
- людей.

Однако по мере развития проекта стало очевидно, что человек состоит не только из мыслей.

Мы постоянно совершаем действия.

Во время действий появляются новые мысли.

Мысли рождают планы.

Планы превращаются в действия.

Действия вызывают воспоминания.

Воспоминания влияют на будущие решения.

Иногда сон становится причиной идеи.

Иногда идея рождается совершенно самостоятельно.

Иногда действие никак не связано ни с чем.

Реальная жизнь человека представляет собой огромную сеть взаимосвязанных объектов.

Именно эту сеть Version 3 пытается сохранить максимально близко к реальности.

---

# Главная идея

Version 3 хранит не выводы.

Version 3 хранит факты.

Например

не

> Пользователь дисциплинирован на 82%.

а

> Пользователь четыре месяца подряд бегал три раза в неделю.

или

> Пользователь ежедневно изучал английский язык.

или

> Пользователь завершил 17 проектов.

Именно из этих фактов различные алгоритмы могут делать выводы.

Сегодня один алгоритм считает дисциплину одним способом.

Через два года может появиться совершенно другой алгоритм.

История пользователя при этом останется неизменной.

Version 3 хранит только объективные факты.

Все выводы являются производными.

# Принципы архитектуры

Version 3 строится вокруг нескольких фундаментальных принципов.

---

## Принцип №1

### Факты никогда не изменяются

После того как событие произошло,

оно становится частью истории жизни человека.

Например

Сегодня была записана мысль.

Завтра искусственный интеллект может полностью изменить свое мнение относительно этой мысли.

Но сама мысль останется неизменной.

История никогда не переписывается.

Меняется только её понимание.

---

## Принцип №2

### Любая сущность существует самостоятельно

Мысль может существовать без причины.

Сон может существовать без причины.

План может существовать без причины.

Действие может существовать без причины.

Никакая сущность не обязана иметь связи.

Связи появляются только тогда,

когда пользователь или алгоритм действительно их обнаружил.

Это делает модель максимально близкой к реальной жизни.

## Принцип №4

### Жизнь представляет собой граф

Version 3 рассматривает жизнь человека как граф.

Не дерево.

Не список.

Не последовательность записей.

Именно граф.

Каждая сущность становится вершиной графа.

Каждая связь становится ребром графа.

Количество связей не ограничено.

Одна мысль может быть одновременно связана

Это полностью соответствует устройству человеческой памяти.

Точно так же

новая активность

не должны требовать изменения фундаментальной модели данных.

Version 3 проектируется как долгоживущая архитектура.

# Graph Model

Version 3 рассматривает жизнь человека как граф.

Любой объект жизни является вершиной графа (Node).

Любая смысловая связь между объектами является ребром графа (Edge).

Именно граф является центральной моделью всей системы.

Все остальные механизмы (AI, статистика, характеристики, RPG, аналитика) работают поверх этого графа.

---

# Внутренний мир

К внутреннему миру относятся сущности,
существующие исключительно в сознании человека.

## Dream

Описание сна.

Не интерпретация.

Не анализ.

Не мнение AI.

Только факт.

Что именно приснилось.

Когда было записано.

Какие детали пользователь захотел сохранить.

---

## Thought

Мысль,
которая появилась в настоящий момент.

Мысль может возникнуть

сама по себе,

во время действия,

после разговора,

после сна,

после чтения книги,

после просмотра фильма,

или вообще без какой-либо известной причины.

Архитектура никак не ограничивает происхождение мысли.

---

## Memory

Memory —

не прошлое.

Memory —

это воспоминание о прошлом.

Это разные вещи.

Два человека могут помнить одно и то же событие совершенно по-разному.

Именно поэтому воспоминание является самостоятельной сущностью.

---

## Plan

План представляет собой намерение совершить некоторое действие в будущем.

План может быть

выполнен,

изменён,

отменён,

забыт,

разделён на несколько действий,

или породить совершенно другой план.

Все подобные сценарии описываются исключительно графом связей.

Action не является мыслью.

Action не является воспоминанием.

Action —

это объективный факт.

Именно Action соединяет внутренний мир человека
с окружающей действительностью.

Количество связей никак не ограничено.

# Практическая архитектура

До этого момента Version 3 описывала предметную модель.

Теперь необходимо определить, как эта модель реализуется в базе данных.

Главная цель заключается не в создании максимально универсальной схемы.

Главная цель заключается в создании архитектуры, которая остается простой, масштабируемой и понятной спустя годы развития проекта.

---

# Layer 1. Graph

Основой всей системы является граф.

Любая сущность представляет собой Node.

Любая смысловая связь представляет собой Edge.

Именно граф является центральной моделью Version 3.

Все остальные подсистемы работают поверх него.

---

# Layer 2. Domain Objects

Несмотря на то что все сущности являются Node графа,

они остаются самостоятельными объектами предметной области.

Например

Dream

Thought

Memory

Plan

Action

Person

Book

Place

Project

имеют собственные таблицы.

Это сознательное решение.

Каждая сущность обладает собственными свойствами,

которые невозможно корректно описать одной универсальной структурой.

Например

Dream содержит

- содержание сна;
- степень осознанности;
- информацию о кошмаре;

- срок выполнения;

- имя;
- дату рождения;
- степень родства.

Book содержит

- автора;
- ISBN;
- количество страниц.

Таким образом Version 3 сочетает преимущества графовой модели и строгой реляционной структуры.

# Layer 6. Characteristics

Characteristic описывает модель личности пользователя.

Например

Discipline

Logic

Memory

Empathy

Creativity

Attention

Endurance

Strength

Diplomacy

Poetry

Humor

Ни одна характеристика не хранится в базе данных как факт.

Она всегда вычисляется.

Источником вычислений являются

Actions,

Dreams,

Thoughts,

Plans,

Measurements,

Relations

и вся история пользователя.

Таким образом факты никогда не меняются,

а способы их интерпретации могут совершенствоваться бесконечно.

---

# Масштабируемость

Version 3 проектируется таким образом,

чтобы новые возможности добавлялись без изменения фундаментальной архитектуры.

Добавление нового типа Activity не требует изменения графовой модели.

Добавление нового Measurement не требует изменения существующих таблиц.

Добавление новой Characteristic не требует изменения истории пользователя.

Добавление новой сущности требует создания новой предметной таблицы,

после чего она автоматически становится частью общего графа.

Это позволяет развивать систему годами,

не разрушая накопленные данные.

# Главный принцип Version 3

Version 2 была системой хранения записей.

Version 3 становится системой моделирования человеческой жизни.

Она не пытается определить,

каким является человек.

Она сохраняет факты его жизни,

их взаимосвязи,

измерения,

контекст

и действия.

Все остальные выводы —

характер,

привычки,

уровень навыков,

паттерны поведения,

эмоциональные закономерности,

психологический анализ,

являются лишь различными способами интерпретации одной и той же истории.

История остается неизменной.

Меняется только глубина её понимания.
